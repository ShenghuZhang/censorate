## 🎯 1. 产品概览

### 1.1 产品定位

**Censorate** 是一个**AI原生的需求管理系统**，通过AI代理自动化需求全流程，帮助团队更高效地管理产品需求从构思到落地的完整生命周期。

### 1.2 核心价值

| 价值点 | 描述 |
|--------|------|
| 🤖 AI驱动 | 内置多个专业AI代理，自动处理需求分析、设计、开发、测试等环节 |
| 📊 可视化流程 | 看板式的泳道管理，直观展示需求状态流转 |
| 🔄 状态机引擎 | 完善的需求状态机，确保流程规范性 |
| 📈 数据分析 | 内置分析仪表板，实时洞察团队产能和交付效率 |
| 🔗 飞书集成 | 与飞书深度集成，支持从飞书消息直接创建需求 |

### 1.3 目标用户

- **产品经理** - 需求规划和管理
- **研发团队** - 需求执行和交付
- **设计团队** - 方案设计和评审
- **测试团队** - 测试用例和质量保证
- **项目经理** - 项目进度和风险管理

## ✨ 3. 核心功能

### 3.1 项目管理

#### 3.1.1 项目创建
- 支持自定义项目名称和描述
- 项目图标上传/Emoji选择
- 工作流配置（预设模板/自定义）

#### 3.1.2 项目设置
- 基本信息管理
- 团队成员管理
- 仓库集成
- 泳道配置

### 3.2 看板管理

#### 3.2.1 泳道配置
提供多种预置工作流模板：
- **Default (4 lanes)**: Backlog → Todo → In Review → Done
- **Standard (6 lanes)**: 需求 → 需求分析 → 方案设计 → 开发 → 测试 → 完成
- **Custom**: 完全自定义的工作流配置

#### 3.2.2 需求卡片
- 需求标题和描述
- 需求状态流转
- 负责人分配
- 完成时间追踪
- AI建议展示
- 退回计数统计

### 3.3 AI代理系统

#### 3.3.1 代理类型

| 代理名称 | 职责描述 |
|----------|----------|
| 🧠 **Analysis Agent** | 需求分析，提取核心信息，识别潜在风险 |
| 🎨 **Design Agent** | 方案设计，提供技术方案和设计建议 |
| 💻 **Development Agent** | 开发指导，代码示例，技术选型建议 |
| 🧪 **Testing Agent** | 测试用例生成，质量检查，测试建议 |

#### 3.3.2 代理执行
- 泳道绑定：每个泳道可配置触发的代理类型
- 自动执行：需求进入特定状态时自动触发代理
- 上下文继承：代理执行时获取历史执行上下文
- 线程对话：支持与代理进行多轮对话

### 3.4 分析仪表板

#### 3.4.1 状态概览
- 各泳道需求数量分布
- 完成率计算（基于最后一个泳道）
- 可视化进度条

#### 3.4.2 每日吞吐量
- 每日完成需求数量统计
- 每日结束时的积压量
- 7天/14天/30天时间范围切换
- 柱状图可视化展示

#### 3.4.3 成员工作负载
- 每个成员的需求分配情况
- 按状态维度的工作分布
- 负载均衡分析

### 3.5 飞书集成

#### 3.5.1 飞书消息创建需求
- 直接从飞书消息提取需求内容
- 自动生成需求标题和描述
- 支持飞书文档链接

#### 3.5.2 飞书文档生成
- 自动为新需求创建飞书文档
- 文档模板化
- 双向链接管理

## 🚀 5. 使用指南

### 5.1 快速开始

#### 步骤1: 创建项目
1. 访问 `/projects` 页面
2. 点击 "New Project" 按钮
3. 填写项目基本信息
4. 选择工作流模板或自定义
5. 保存并进入看板

#### 步骤2: 添加需求
1. 在看板页面点击添加需求
2. 填写需求标题和描述
3. 保存需求（自动进入Backlog）

#### 步骤3: 流转需求
1. 拖拽需求卡片到目标泳道
2. 如配置了AI代理，等待代理执行
3. 查看AI建议并继续流转
4. 到达完成泳道即标记完成

### 5.2 工作流最佳实践

#### 非技术团队 (4个泳道)
```
Backlog → Todo → In Review → Done
```

#### 技术团队 (6个泳道)
```
需求 → 需求分析 → 方案设计 → 开发 → 测试 → 完成
```

### 5.3 AI代理配置

#### 配置代理触发
1. 进入项目设置
2. 选择 "泳道配置"
3. 为特定泳道选择代理类型
4. 配置代理参数（可选）
5. 保存配置

#### 代理使用技巧
- 退回计数：AI会关注需求被退回的次数
- 上下文：代理会获取该需求的历史执行记录
- 线程对话：可以与代理进行多轮沟通

## 📊 6. 数据分析

### 6.1 关键指标

| 指标 | 定义 | 用途 |
|------|------|------|
| **完成率** | 已完成需求 / 总需求 | 项目整体进度 |
| **吞吐量** | 每日完成需求数 | 团队产能变化 |
| **积压量** | 每日结束时未完成需求 | 交付压力评估 |
| **退回率** | 被退回过的需求占比 | 流程/质量问题识别 |

### 6.2 分析方法

#### 趋势分析
- 观察吞吐量曲线，识别产能变化
- 对比积压量与完成量的关系
- 识别团队交付周期

#### 负载分析
- 查看成员工作负载分布
- 识别资源瓶颈
- 优化任务分配

## 🔌 7. API文档

### 7.1 认证

采用Bearer Token认证方式：
```http
Authorization: Bearer <your_token>
```

### 7.2 核心接口

#### 项目管理
- `GET /api/v1/projects` - 获取项目列表
- `POST /api/v1/projects` - 创建项目
- `GET /api/v1/projects/{id}` - 获取项目详情
- `PUT /api/v1/projects/{id}` - 更新项目

#### 需求管理
- `GET /api/v1/projects/{id}/requirements` - 获取项目需求列表
- `POST /api/v1/projects/{id}/requirements` - 创建需求
- `GET /api/v1/requirements/{id}` - 获取需求详情
- `PUT /api/v1/requirements/{id}` - 更新需求
- `POST /api/v1/requirements/{id}/transition` - 需求状态流转

#### 分析数据
- `GET /api/v1/projects/{id}/analytics` - 获取项目统计
- `GET /api/v1/projects/{id}/analytics/throughput` - 获取吞吐量数据
- `GET /api/v1/projects/{id}/analytics/workload` - 获取工作负载数据

### 7.3 代理执行
- `POST /api/v1/requirements/{id}/agent-execute` - 手动触发代理执行
- `GET /api/v1/requirements/{id}/agent-executions` - 获取代理执行历史

## 🔧 8. 部署指南

### 8.1 环境要求

#### 前端环境
- Node.js 18+
- npm 或 yarn

#### 后端环境
- Python 3.9+
- SQLite 或 PostgreSQL

### 8.2 快速部署

#### 开发环境
```bash
# 克隆项目
git clone <repo_url>
cd censorate-system

# 启动服务
./run.sh start
```

#### 生产环境
```bash
# 后端配置
cd backend
cp .env.example .env
# 编辑 .env 配置

# 数据库迁移
# 使用 Alembic 迁移

# 启动服务
gunicorn main:app --workers 4

# 前端构建
cd ../frontend
npm run build
npm start
```

### 8.3 配置项

#### 后端环境变量
| 变量名 | 描述 | 默认值 |
|--------|------|--------|
| `DATABASE_URL` | 数据库连接字符串 | sqlite:///./censorate.db |
| `SECRET_KEY` | JWT密钥 | (自动生成) |
| `LARK_APP_ID` | 飞书应用ID | - |
| `LARK_APP_SECRET` | 飞书应用密钥 | - |
| `DEEPAGENT_API_KEY` | DeepAgent API密钥 | - |

#### 前端环境变量
| 变量名 | 描述 | 默认值 |
|--------|------|--------|
| `NEXT_PUBLIC_API_URL` | 后端API地址 | http://localhost:8216/api/v1 |

## 🆘 10. 常见问题

### Q: 如何重置数据库？
A: 删除后端目录下的 `censorate.db` 文件，重启服务会自动重新创建。

### Q: AI代理没有触发怎么办？
A: 检查：1) 泳道是否正确配置了代理 2) API密钥是否正确配置 3) 代理状态是否正常

### Q: 如何备份数据？
A: 直接复制 `censorate.db` 文件即可；使用PostgreSQL的话请使用标准数据库备份工具。

### Q: 支持多语言吗？
A: 当前版本主要支持中文和英文界面，后续版本会完善多语言支持。
