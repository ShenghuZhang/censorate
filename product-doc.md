## 产品概述

<callout emoji="🚀" background-color="light-blue">
Censorate 是一个 AI 原生的需求管理系统，通过 AI 代理自动化需求全流程，帮助团队更高效地管理产品需求从构思到落地的完整生命周期。
</callout>

### 核心价值

| 价值点 | 描述 |
|--------|------|
| 🤖 **AI 驱动** | 内置多个专业 AI 代理，自动处理需求分析、设计、开发、测试等环节 |
| 📊 **可视化流程** | 看板式的泳道管理，直观展示需求状态流转 |
| 🔄 **状态机引擎** | 完善的需求状态机，确保流程规范性 |
| 📈 **数据分析** | 内置分析仪表板，实时洞察团队产能和交付效率 |
| 🔗 **飞书集成** | 与飞书深度集成，支持从飞书消息直接创建需求 |

### 目标用户

- **产品经理** - 需求规划和管理
- **研发团队** - 需求执行和交付
- **设计团队** - 方案设计和评审
- **测试团队** - 测试用例和质量保证
- **项目经理** - 项目进度和风险管理

## 核心功能模块

### 1. 看板管理 (Kanban)

看板是 Censorate 的核心功能，提供看板式的泳道管理。

<grid cols="2">
<column>

#### 主要特性
- 多泳道工作流配置
- 需求卡片拖拽流转
- AI 代理自动触发
- 需求详情查看和编辑
- 负责人分配
- 优先级标记

</column>
<column>

#### 预设工作流
- **Default (4 泳道)
- **Standard (6 泳道)**
- **Custom (自定义)**

</column>
</grid>

---

### 2. 分析仪表板 (Analytics)

提供全面的数据分析功能，帮助团队了解项目进度和产能。

#### 核心指标
- **状态概览** - 各泳道需求数量分布和完成率
- **每日吞吐量** - 每日完成需求数量统计和趋势
- **成员工作负载** - 每个成员的需求分配情况

---

### 3. AI 代理系统 (Agents)

内置多个专业 AI 代理，自动化处理需求的各个环节。

| 代理名称 | 职责描述 |
|----------|----------|
| 🧠 **Analysis Agent** | 需求分析，提取核心信息，识别潜在风险 |
| 🎨 **Design Agent** | 方案设计，提供技术方案和设计建议 |
| 💻 **Development Agent** | 开发指导，代码示例，技术选型建议 |
| 🧪 **Testing Agent** | 测试用例生成，质量检查，测试建议 |

#### 代理执行特性
- 泳道绑定：每个泳道可配置触发的代理类型
- 自动执行：需求进入特定状态时自动触发代理
- 上下文继承：代理执行时获取历史执行上下文
- 线程对话：支持与代理进行多轮对话

---

### 4. 技能管理 (Skills)

管理 AI 代理的技能定义，支持自定义技能上传和管理。

- 技能分类浏览
- 技能版本管理
- 自定义技能上传
- 技能下载和分享

---

### 5. 项目设置 (Settings)

项目配置和管理功能。

#### 项目配置
- 基本信息管理（名称、描述、图标）
- 团队成员管理
- 工作流/泳道配置
- 仓库集成

### 后端架构分层

```
┌─────────────────────────────────────────────────┐
│          API 层 (app/api/v1/endpoints)      │
├─────────────────────────────────────────────────┤
│          服务层 (app/services)              │
├─────────────────────────────────────────────────┤
│          代理层 (app/agents)                │
├─────────────────────────────────────────────────┤
│          仓库层 (app/repositories)          │
├─────────────────────────────────────────────────┤
│          模型层 (app/models)                │
└─────────────────────────────────────────────────┘
```

#### 核心模块说明

| 模块 | 职责 |
|------|------|
| **API 层** | 提供 RESTful API 接口，处理 HTTP 请求和响应 |
| **服务层** | 实现业务逻辑，协调各组件工作 |
| **代理层** | AI 代理执行引擎，管理代理生命周期和技能调用 |
| **仓库层** | 数据访问抽象，提供统一的数据操作接口 |
| **模型层** | SQLAlchemy ORM 模型定义 |

---

### AI 代理架构

```
┌─────────────────────────────────────────┐
│         Agent Registry               │
│  （代理注册和发现）                  │
├─────────────────────────────────────────┤
│         Base Agent                   │
│  （代理基类，提供公共能力）          │
├─────────────────────────────────────────┤
│    ┌────────────┬────────────┐    │
│    │ Analysis   │  Design    │    │
│    │   Agent    │   Agent    │    │
│    └────────────┴────────────┘    │
│    ┌────────────┬────────────┐    │
│    │Development │  Testing   │    │
│    │   Agent    │   Agent    │    │
│    └────────────┴────────────┘    │
├─────────────────────────────────────────┤
│         Skill System               │
│  （技能加载和执行）              │
└─────────────────────────────────────────┘
```

#### 代理与技能的关系

- **代理**：具有特定领域能力的 AI 助手
- **技能**：代理可执行的具体任务能力
- **关系**：一个代理可以拥有多个技能，一个技能可以被多个代理使用

## 快速开始指南

### 1. 创建项目
1. 访问项目管理页面
2. 点击 "New Project" 按钮
3. 填写项目基本信息
4. 选择工作流模板或自定义
5. 保存并进入看板

### 2. 添加需求
1. 在看板页面点击添加需求
2. 填写需求标题和描述
3. 保存需求（自动进入 Backlog）

### 3. 流转需求
1. 拖拽需求卡片到目标泳道
2. 如配置了 AI 代理，等待代理执行
3. 查看 AI 建议并继续流转
4. 到达完成泳道即标记完成
